# QA_CHECKLIST.md

## Antes de entregar cambios

- [ ] `npm test` pasa.
- [ ] `npm run build` pasa o se documenta por qué no se pudo ejecutar.
- [ ] El juego arranca con `npm run dev`.
- [ ] Mouse funciona.
- [ ] Touch/móvil navegador funciona o se indica pendiente.
- [ ] Se puede limpiar sala 1.
- [ ] Se puede recibir daño.
- [ ] Se puede morir.
- [ ] Se puede elegir mejora.
- [ ] Se puede avanzar de sala.
- [ ] No hay errores obvios en consola.

## Checklist de diversión

- [ ] Lanzar al héroe se siente satisfactorio.
- [ ] El jugador entiende cuándo puede actuar.
- [ ] Los peligros son legibles.
- [ ] Los barriles son útiles.
- [ ] Los proyectiles tienen razón de existir.
- [ ] Hay al menos una decisión interesante por sala.
